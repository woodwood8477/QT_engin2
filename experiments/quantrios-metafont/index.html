This preview is a browser-drawn sketch of the METAFONT idea: glyphs are treated as centerline strokes before becoming dense SVG outlines. The actual source lives in QuanTrios.mf.

u#1pt
stem#.46pt
cap#8pt
xh#5pt
% QuanTrios.mf
mode_setup;
font_identifier := "QuanTriosMeta";
font_size 10pt#;

u# := 1pt#;
stem# := .46pt#;
cap# := 8pt#;
xh# := 5pt#;
desc# := 1.2pt#;

define_pixels(u, stem, cap, xh, desc);

def qt_pen =
  pickup pencircle scaled stem
enddef;

beginchar("Q", 8u#, cap#, desc#);
  qt_pen;
  draw fullcircle
    xscaled 6u
    yscaled .94cap
    shifted (4u, .50cap);
  draw (5.35u, .18cap) -- (7.15u, -0.72u);
endchar;

beginchar("T", 7u#, cap#, 0);
  qt_pen;
  draw (.70u, cap) -- (6.30u, cap);
  draw (3.50u, cap) -- (3.50u, 0);
endchar;

Next implementation direction

Keep this METAFONT source as a reference specimen. The next QT-engine step should be a JavaScript centerline renderer that uses the same skeletal glyph logic, then maps audio parameters to width, aperture, tension, and motion. That avoids forcing dense SVG outlines into deformation.
